Major Refactoring (December 2025)

Based on instructor feedback, I worked with AI to comprehensively refactor the entire project for a mastery-level grade. Here's what changed:

HTML Improvements

  • Description Shortening: AI helped rewrite all 12 button data-desc attributes from essay-length paragraphs (10+ sentences) to concise 2-4 sentence descriptions that are easier to scan in the UI
  • Evaluation: I reviewed each shortened description to ensure it retained the key information while being more user-friendly. For example, the Fouls description went from 10+ sentences to just 3 focused sentences about personal, technical, and flagrant fouls.

CSS Fixes and Optimizations

  • Contrast Improvements: AI helped identify and fix accessibility issues by darkening text from #1a1a1a to #0f1419 and lightening backgrounds to #e5b89a for better readability
  • Media Query Repair: AI located a critical bug at line 252 where a stray closing brace was breaking the entire responsive layout. Fixed by properly wrapping all mobile styles in the @media query block.
  • CSS Deduplication: AI identified repeated rules (.court-caption defined twice, .court-image.show defined twice) and consolidated them to eliminate redundancy
  • Animation Simplification: AI helped unify conflicting animations (bounce, pass, slide, scale) into a single basketball bounce theme that's cohesive and professional
  • Evaluation: I tested the contrast changes with browser dev tools to verify improved readability. I tested responsive layout on mobile to confirm the media query fix resolved the broken mobile view. The simplified animations now feel polished instead of chaotic.

JavaScript Refactoring

  • Function Extraction: AI helped refactor repetitive code by creating helper functions:
    • activateButton(btn) - handles highlighting the clicked button
    • applyTitleBounce() - triggers the title bounce animation
    • showDescriptionOnly(desc) - displays text-only for rule buttons
    • showImageMode(file, desc) - displays court image with description
    • isRuleButton(btn) - checks if button is a text-only rule button
    • getImageFile(btn) - determines which court image to display
  • DOMContentLoaded Consolidation: AI helped merge 3 separate initialization blocks into a single unified setup function, reducing code from 379 lines to 230 lines
  • Mobile Support: AI added click fallback to dropdowns so they work on both hover (desktop) and click (mobile/touch devices)
  • Preloading Simplification: AI removed the complex 60-line preloading logic since modern browsers and the simple image set load quickly without it
  • Evaluation: The refactored code is much easier to read and understand. Each function has a clear purpose with descriptive comments. I tested the mobile dropdown functionality on touch devices to confirm it works correctly. The simplified code maintained all original functionality while being more maintainable.
